# Workflow Tool Spec

This folder turns the Workflow OS doctrine into a buildable internal product.

It is a spec for a local-first tool that helps a solo founder or technical operator:

- dump raw daily notes
- turn them into structured tasks
- route tasks to the right model/tool
- generate efficient prompts
- track what should be done now vs later
- reduce wasted premium model usage

## Included

- [`product-spec.md`](product-spec.md) — full product spec
- [`mvp-build-plan.md`](mvp-build-plan.md) — smallest credible MVP and build order
- [`seeded-examples.md`](seeded-examples.md) — example note transformations and routing decisions

## Relationship to Workflow OS

- `workflow-os/` = doctrine and operating model
- `workflow-tool-spec/` = the internal app that operationalizes that model

This makes the repo useful both as a thinking system and as a buildable product blueprint.
