# Workflow OS

This folder contains a practical **operating system for AI-assisted development** built around:

- **Cursor** as the editor and control center
- **Codex** as the primary builder
- **Claude Code** as the debugger, reviewer, and planner

The goal is simple: get more output from the same tools while wasting fewer premium requests.

## What is included

- [`workflow-operating-system.md`](workflow-operating-system.md) — the main workflow doctrine
- [`daily-execution-template.md`](daily-execution-template.md) — a daily planning template for raw notes, routing, and execution
- [`prompt-routing-templates.md`](prompt-routing-templates.md) — reusable prompt structures for build, debug, review, and task decomposition

## Core rules

- **Codex builds**
- **Claude diagnoses and reviews**
- **Cheap/local models handle low-value transforms**
- **Cursor is the control center, not the brain**
- **Never waste frontier model usage on vague tasks**

## Use this folder when

- you are hitting usage limits too quickly
- you want tighter prompts and less rework
- you want a repeatable system for daily engineering execution
- you need to turn messy notes into scoped work units

## Outcome

This is not theory. It is a reusable operating layer for:

- routing tasks correctly
- reducing prompt waste
- separating diagnosis from implementation
- enforcing acceptance criteria
- improving daily execution quality
